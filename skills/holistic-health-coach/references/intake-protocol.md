# 摄入协议 / Intake Protocol

目标：用最少的提问拿到能改变结论的数据，其余从已知信息推断或标【未知】。

## 分批提问（按对结论的影响排序）

**第 1 批（几乎总要问，决定大方向）**
- 性别、年龄、身高、体重（→ BMI/TDEE/蛋白质目标）。
- 最想优先解决的 1–2 个问题 + 起始时间 + 趋势。
- 睡眠（时长、入睡/起床时间）。
- 现有补剂 + 在用药物（查重复/相互作用/过量，安全优先）。

**第 2 批（细化机制假设）**
- 典型一日饮食 + 进食时间窗 + 高频嗜好（糖/奶/油炸/外卖）。
- 屏幕/短视频/游戏时长，使用场景（睡前？吃饭时？）。
- 运动、久坐、日晒、压力、咖啡因/酒。

**第 3 批（按需，验证假设）**
- 体检/血检报告（见下「高价值指标」）。
- 具体症状细节（痘的部位/类型、记忆问题的具体表现等）。
- 家族史、过敏史。

> 原则：每次最多问 3–6 个高价值问题；用户嫌烦就先用已有信息给初步分析，再逐步补全。

## 高价值客观指标（建议用户去测，能大幅减少猜测）
- **代谢**：空腹血糖、HbA1c、空腹胰岛素（算 HOMA-IR）、血脂四项。
- **营养**：铁蛋白 + 转铁蛋白饱和度、25-OH 维生素D、维生素B12、（女性）必要时叶酸。
- **激素/甲状腺**：TSH（必要时 fT3/fT4）、（痘/多毛女性）雄激素、（疲劳男性）睾酮。
- **炎症**：hs-CRP。
- **肝肾功能**：补剂安全基线。

## 数据卫生
- 单位务必明确（kg/lb、cm/in、mg/g/IU/mcg）。换算别口算，用脚本或显式计算。
- 区分「用户主观描述」与「客观测量」，推理时分别对待。
- 缺失就标【未知】并说明它如何影响结论的确定性。
- 所有数据写回 `health-profile.md`，敏感信息仅本地保存，不外传。
