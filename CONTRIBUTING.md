# 贡献指南 / Contributing

感谢你想让这个私人健康管家变得更好！这是一个会直接影响人们健康决策的项目，所以我们对**准确性和证据**要求较高。

## 核心原则

1. **所有健康主张都要有据可查**。新增或修改任何关于饮食/补剂/运动/皮肤/睡眠的结论时，必须：
   - 标注**证据强度** `[A/B/C/D]`（定义见 `skills/holistic-health-coach/references/evidence-and-safety.md`）。
   - 尽量给出**可核查的权威来源**（系统综述/meta 分析 > RCT > 观察性 > 机制/动物）。优先 WHO、各国指南、NIH ODS、Cochrane、UpToDate、专科学会（AAD/AASM 等）。
   - **不要**引用不存在或无法核查的论文。拿不准就标「需进一步查证」。
2. **不夸大、不贩卖焦虑、不推销品牌或具体商品链接**。
3. **安全优先**。涉及剂量、相互作用、禁忌时必须保守，并保留「就医分流」的安全红线。
4. **区分相关与因果**。多数营养/生活方式证据是观察性的，措辞要诚实。
5. **隐私**。绝不在 PR / issue / 示例里包含任何真实个人健康数据。示例一律用虚构人物。

## 怎么改

- 修改知识内容 → 编辑对应 `references/*.md`；保持「做什么 / 机制 / 量化 / 证据」的风格。
- 调整主流程/工作流 → 编辑 `SKILL.md`（保持 < 500 行，细节放 references）。
- 新增专题（如「肠道健康」「女性周期」「压力管理」）→ 新建 `references/<topic>.md` 并在 `SKILL.md` 资源索引里登记。
- 新增人群考量 → 改 `references/population-considerations.md`。
- 改脚本 → 确保 `python scripts/bmi_tdee.py ...` 能跑通。

## 提交流程

1. Fork & 新建分支。
2. 改动尽量聚焦、附上来源链接（在 PR 描述里列出引用）。
3. 自检：SKILL.md 的 frontmatter 合法、目录名与 `name` 一致、脚本可运行、没有泄露隐私数据。
4. 提 PR，说明改了什么、依据是什么、证据强度。

## 翻译

欢迎把 skill 内容翻译成英文/其他语言（建议放在 `skills/holistic-health-coach/references/<lang>/` 或独立的语言分支），便于全球用户使用。

## 行为准则

请保持友善、就事论事。健康话题敏感，讨论时尊重彼此、以证据为准。
